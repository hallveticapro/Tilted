## Architecture Overview

Tilted is a single-page static React app. `src/App.tsx` owns screen routing through local state rather than a router package. Built-in decks are bundled at build time from `src/data/`, and custom decks are validated and stored locally through `src/services/deckStorage.ts`.

There is no server/API code, auth layer, database, CORS configuration, backend logging, or analytics integration. Persistence is browser-origin LocalStorage for custom decks, preferences, favorites, and round history. Team sessions are in memory during play, but completed round results are saved to local history.

Deployment supports static hosts, Vercel/Netlify-style security headers, and Docker. Docker builds the Vite app with Node 22, then serves the static output from pinned unprivileged Nginx on port 8080. Runtime metadata injection copies the built files into `/tmp/tilted-html` so a read-only root filesystem can still receive environment-specific social metadata.

### Medium

#### MED-001 - Production service worker is cached for four hours

- Severity: Medium
- Category: Deployment / PWA reliability
- Location: `deploy/nginx.conf:22-25`, production `https://tilted.mrhallsclass.com/sw.js`
- Evidence: Repo Nginx config sets `expires -1` for `/sw.js`, but `curl --head https://tilted.mrhallsclass.com/sw.js` returned `Cache-Control: max-age=14400` on 2026-06-04.
- Impact: New service-worker fixes, offline cache revisions, or update-banner behavior may be delayed for users by the proxy/CDN cache.
- Recommendation: Adjust Cloudflare/reverse-proxy rules so `/sw.js` is `no-cache` or must-revalidate, then verify with `curl --head` after deployment.
- Suggested priority: Immediate

#### MED-002 - Team/player names can persist in local round history

- Severity: Medium
- Category: Classroom privacy / shared-device safety
- Location: `src/components/GameScreen.tsx:42-60`, `src/services/roundHistory.ts:38-45`, `src/components/HistoryScreen.tsx:21-24`
- Evidence: Round results include `teamName` and `playerName`; `saveRoundHistory` serializes the full result to LocalStorage; the history screen displays the saved team name. CSV export is anonymous by default, but persisted history is not stripped.
- Impact: If a teacher enters real student names on a shared classroom device, those names can remain visible to later users of the same browser origin until history is cleared.
- Recommendation: Add a privacy-first history option that omits `playerName` and possibly `teamName` from persisted records by default, or prompt teachers to choose whether names should be saved. Consider auto-clearing team-player names after a session while preserving anonymous scores.
- Suggested priority: Immediate

#### MED-003 - Production social metadata points at GitHub defaults

- Severity: Medium
- Category: Deployment / sharing
- Location: `vite.config.ts:8-11`, `deploy/40-runtime-metadata.sh:6-18`, production root HTML
- Evidence: Production HTML currently has `og:url` as `https://github.com/hallveticapro/Tilted` and `og:image`/Twitter image as the raw GitHub cover URL. `TASKS.md` names `https://tilted.mrhallsclass.com` as production.
- Impact: Shared links can advertise the repository instead of the live classroom app, and self-hosted metadata may look less polished or confusing.
- Recommendation: Set `TILTED_PUBLIC_URL=https://tilted.mrhallsclass.com` and a production-hosted `TILTED_SHARE_IMAGE_URL` in the runtime environment, or bake equivalent `VITE_*` values into static deployments.
- Suggested priority: Immediate

#### MED-004 - Production HSTS preload policy needs domain-wide confirmation

- Severity: Medium
- Category: Deployment / operational security
- Location: Production response headers; README HSTS guidance
- Evidence: `curl --head https://tilted.mrhallsclass.com` returned `Strict-Transport-Security: max-age=63072000;includeSubDomains; preload`. The README recommends adding HSTS after HTTPS is stable and avoiding `preload` until every relevant subdomain is permanently HTTPS-ready.
- Impact: `includeSubDomains; preload` affects the broader domain, not only Tilted. If any subdomain under `mrhallsclass.com` is not HTTPS-ready, preload can create access problems.
- Recommendation: Confirm this policy is intentional for the entire domain. If it is not, remove `preload` and/or `includeSubDomains` at the TLS-terminating proxy for this host.
- Suggested priority: Next

### Low

#### LOW-001 - Projected card prompts inherit negative heading letter spacing

- Severity: Low
- Category: Accessibility / classroom readability
- Location: `src/styles/global.css:47-52`, inherited by `.game-card h1`
- Evidence: Global `h1` sets `letter-spacing: -0.065em`; game prompt headings do not override it.
- Impact: Large prompt text may look tighter on classroom projectors or older displays, especially for long custom prompts and younger readers.
- Recommendation: Set game prompt letter spacing to `0` and visually verify common long prompts on mobile landscape and projected desktop widths.
- Suggested priority: Next

#### LOW-002 - Service-worker cache revisioning is manual

- Severity: Low
- Category: Maintainability / offline reliability
- Location: `public/sw.js:1-24`, `public/sw.js:52-64`
- Evidence: `CACHE_NAME` is manually set to `tilted-shell-v3`; runtime-cached requests are stored in the same named cache until the name changes.
- Impact: If future shell changes forget to bump the cache name, obsolete runtime entries may accumulate and update debugging becomes harder.
- Recommendation: Generate the cache name from a build/version value or add explicit runtime-cache pruning for hashed assets.
- Suggested priority: Later

#### LOW-003 - Shared deck links expose full deck content in copied URLs

- Severity: Low
- Category: Privacy / sharing UX
- Location: `src/services/deckSharing.ts:19-22`, `src/components/DeckEditor.tsx:340-358`
- Evidence: `createDeckShareUrl` encodes the exported deck JSON into the URL fragment; the editor shares or copies that URL.
- Impact: This avoids a backend and keeps fragments out of normal HTTP requests, but copied links can be long and reveal every custom prompt/hint to anyone who receives or sees the URL.
- Recommendation: Add concise UI copy before sharing that says the link contains the full deck content. Consider QR export or file-based sharing for large or sensitive classroom decks.
- Suggested priority: Later
