# Tilted Web App Audit - 2026-06-01

## Scope

This audit reviews the current `main` iteration of Tilted as a public, static React game deployed through Docker and GHCR. It covers player flow, motion controls, deck management, accessibility, dependency risk, container defaults, reverse-proxy behavior, metadata, and maintainability.

The current iteration is in good shape for a small public web app: the production build succeeds, the automated tests pass, the game already randomizes cards at the start of each round, and the container runs an unprivileged Nginx image. The recommendations below focus on practical improvements rather than a large rewrite.

## Priority Findings

### 1. Upgrade Vitest to remove a critical development dependency advisory

**Priority:** High  
**Area:** Security / developer tooling

`npm audit --json` reports one critical advisory against the resolved `vitest@3.2.6` dependency: [GHSA-5xrq-8626-4rwp](https://github.com/advisories/GHSA-5xrq-8626-4rwp). The affected range is `<4.1.0`, and npm currently recommends `vitest@4.1.8`.

The advisory concerns Vitest's optional UI server, so the deployed static Tilted bundle is not directly exposed. It still matters for contributors and local development, especially if a dev or test UI server is ever bound to a LAN interface.

**Recommendation:** Upgrade Vitest to `4.1.8` or a newer compatible release, run the full test suite, and keep Vite/Vitest development servers bound to localhost unless LAN access is intentional.

### 2. Let players continue immediately when motion permission fails

**Priority:** High  
**Area:** Functionality / mobile UX  
**Evidence:** `src/App.tsx:121-136`

Starting a motion-enabled round only advances when `requestPermission()` returns `true`. If permission is denied, unavailable, or errors, the player remains on setup and must discover that they need to disable motion manually.

That is recoverable, but it feels like the Start button did nothing. Motion controls should enhance gameplay without blocking it.

**Recommendation:** After a failed motion request, show a clear inline message and a prominent `Continue with buttons` action. Consider automatically switching the current round to touch controls after the player acknowledges the fallback. Preserve the option to retry motion from setup later.

### 3. Bound and salvage imported custom deck data

**Priority:** High  
**Area:** Data safety / robustness  
**Evidence:** `src/services/deckStorage.ts:22-88`

Custom deck validation checks basic types, but it does not enforce limits for imported JSON size, deck count, cards per deck, or string lengths. A very large import can consume browser storage quota and make the editor sluggish.

The storage loader also catches a malformed saved payload and returns an empty array for the entire custom library. A single bad deck can therefore hide otherwise valid decks until storage is repaired manually.

**Recommendation:** Add explicit limits, catch quota errors with a helpful message, validate decks independently, preserve valid decks, and report skipped invalid entries. Keep a recovery option that exports or clears malformed local data.

### 4. Ship browser security headers with the container

**Priority:** Medium  
**Area:** Security / deployment  
**Evidence:** `deploy/nginx.conf:1-25`

The current Nginx configuration handles cache behavior but does not set browser hardening headers. A reverse proxy can add these, but the public container should provide secure defaults too.

**Recommendation:** Add and test:

```nginx
add_header X-Content-Type-Options "nosniff" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header X-Frame-Options "DENY" always;
add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;
```

Add a Content Security Policy after verifying the social preview image, manifest, and Device Orientation behavior on iOS and Android. Be deliberate with motion-related permissions policy directives so the tilt controls continue to work behind the reverse proxy.

### 5. Add pull-request CI and protect `main`

**Priority:** Medium  
**Area:** Release safety / supply chain  
**Evidence:** `.github/workflows/publish-ghcr.yml:23-56`

The existing workflow publishes the container, but the repository does not currently have branch protection enabled for `main`. A GitHub API check returned `Branch not protected`.

The Docker build runs the production build, but it does not run the Vitest suite. A regression can reach GHCR if it compiles successfully.

**Recommendation:** Add a pull-request workflow that runs:

```bash
npm ci
npm run typecheck
npm test -- --run
npm run build
```

Require that workflow on `main`, enable branch protection, and add Dependabot or Renovate for npm and GitHub Actions updates. Pin GitHub Actions and container base images to immutable digests where practical.

## Player Experience

### 6. Remove card totals from player-facing deck tiles

**Priority:** Medium  
**Area:** UX  
**Evidence:** `src/components/DeckSelectScreen.tsx:19-20`

Each playable deck tile currently displays its number of cards. Players do not need that implementation detail when choosing a category, and removing it will make the cards less visually busy.

**Recommendation:** Remove the `<small>{deck.cards.length} cards</small>` line from player-facing deck tiles. Also replace the custom-deck fallback description `${deck.cards.length} cards ready to play.` with neutral copy such as `Your custom guessing deck.`

Keep card totals inside the deck editor because authors need them while managing content. The category-level `N decks` label at `src/components/DeckSelectScreen.tsx:65` is separate; it can remain unless a cleaner selection screen is preferred.

### 7. Make the category scroller arrows stateful and accessible

**Priority:** Medium  
**Area:** UX / accessibility  
**Evidence:** `src/components/CategoryScroller.tsx:20-60`

The new left and right arrows correctly hint that categories can scroll, but both remain active at the beginning and end of the strip. Category selection is represented visually without an explicit pressed state for assistive technology.

**Recommendation:** Track the scroll position, disable or hide arrows that cannot scroll farther, update state on scroll and resize, and expose the selected category with `aria-pressed`. A subtle edge fade can reinforce that more categories are available.

### 8. Add persistent sound and vibration preferences

**Priority:** Medium  
**Area:** Functionality / classroom usability  
**Evidence:** `src/components/GameScreen.tsx:110-113`

Correct and pass feedback now use audio and vibration automatically. That is useful for a host, but classrooms and quiet environments need a fast way to mute sound or disable haptics.

**Recommendation:** Add `Sound effects` and `Vibration` toggles in round setup, persist them locally, and default them on. Respect the vibration setting before calling `navigator.vibrate`.

### 9. Add confirmation or undo for destructive deck actions

**Priority:** Medium  
**Area:** Data safety / touch UX  
**Evidence:** `src/components/DeckEditor.tsx:91-115`

Deleting a custom deck or card happens immediately. On a phone, an accidental tap can remove content that only exists in LocalStorage.

**Recommendation:** Add a confirmation step for deleting a deck and a short undo action for card deletion. Keep JSON export easy to find as the backup path.

### 10. Remove the extra CSS transform from the hero logo

**Priority:** Medium  
**Area:** Visual polish  
**Evidence:** `src/styles/global.css:197-203`

The home-page `.hero-panel__logo` artwork already includes its own tilt. Applying an additional
`transform: rotate(-7deg);` partially straightens the visible mark instead of reinforcing the theme.

**Recommendation:** Remove the CSS transform and let the artwork provide the intended tilt.

## Efficiency And Maintainability

### 11. Avoid serializing every custom deck on each editor keystroke

**Priority:** Medium  
**Area:** Efficiency  
**Evidence:** `src/components/DeckEditor.tsx:62-77`, `src/services/deckStorage.ts:84-88`

Every text edit calls `saveCustomDecks`, serializing the full custom deck collection into LocalStorage. This is acceptable with a few small decks but becomes noticeable after several copied 50-card decks.

**Recommendation:** Update React state immediately for a responsive editor, then debounce LocalStorage persistence or save on blur. Consider memoizing individual card editors so typing in one card does not rerender the full list unnecessarily.

### 12. Clear the final-card completion timeout during cleanup

**Priority:** Low  
**Area:** Reliability  
**Evidence:** `src/components/GameScreen.tsx:121-135`

The feedback timeout is tracked and cleared during unmount. The separate timeout that finishes a round after the final card is not tracked. It can fire after the screen has been left through another action.

**Recommendation:** Store the completion timeout in a ref and clear it with the existing cleanup logic.

### 13. Optimize the SVG favicon and cache root-level static assets

**Priority:** Low  
**Area:** Page weight / caching  
**Evidence:** `public/favicon.svg`, `deploy/nginx.conf:13-16`

The SVG favicon is unusually large for an icon, and the long-lived cache rule only targets `/assets/`. Root-level favicon and manifest files do not receive the same immutable caching treatment.

**Recommendation:** Run the SVG through an optimizer such as SVGO or replace it with a simplified vector. Add explicit caching for versioned favicon and manifest assets, while keeping `index.html` uncached.

### 14. Make social metadata portable for forks and self-hosted instances

**Priority:** Low  
**Area:** Deployment / sharing  
**Evidence:** `index.html:24-45`

The social preview image currently points to a repository-hosted raw GitHub URL. That works for the canonical deployment, but forks and private self-hosted instances will continue advertising the original image URL. The page also does not publish a canonical `og:url`.

**Recommendation:** Allow a deployment URL and share-image URL to be injected at build time, add `og:url`, and document the environment variables. Keep sensible defaults for a zero-configuration Docker deployment.

### 15. Consider optional container hardening after validation

**Priority:** Low  
**Area:** Container security  
**Evidence:** `Dockerfile:11-19`, `docker-compose.yml:9-11`

The runtime already uses `nginxinc/nginx-unprivileged` and `no-new-privileges:true`, which is a good baseline.

**Recommendation:** Test a read-only root filesystem, required `tmpfs` mounts for Nginx runtime files, and dropping any unnecessary capabilities. Document a release-tag or digest pin for Unraid users who prefer controlled updates over the mutable `latest` tag.

## Future Improvements

### 16. Add offline support for classroom reliability

**Priority:** Low  
**Area:** Functionality

Tilted is a static app with built-in decks, so it is a strong candidate for a small service worker. Offline support would make classroom use more resilient when Wi-Fi is unreliable.

**Recommendation:** Cache the app shell and built-in bundle with a versioned update strategy. Do this after the current mobile gameplay polish so stale-cache behavior is tested carefully.

### 17. Review public deck naming and trademark messaging

**Priority:** Low  
**Area:** Public-release hygiene

The public built-in library includes categories and prompts derived from recognizable entertainment franchises. That is useful gameplay content, but a public repository should make its unofficial nature clear.

**Recommendation:** Review the deck list, add a brief unofficial/fan-made disclaimer where appropriate, and keep branding generic so the project is not presented as affiliated with rights holders.

## Suggested Implementation Order

1. Upgrade Vitest and add PR CI with branch protection.
2. Add motion-permission fallback behavior.
3. Remove player-facing card totals and improve category scroller arrow states.
4. Add LocalStorage import limits, salvage behavior, and delete protection.
5. Add sound and vibration preferences.
6. Add Nginx security headers and validate them behind the reverse proxy on iOS and Android.
7. Debounce editor persistence and complete the lower-priority deployment polish.
